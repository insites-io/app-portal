# Contributing

Thanks for your interest in contributing. We accept pull requests under the
[Apache License, Version 2.0](LICENSE).

## Developer Certificate of Origin (DCO)

We use the [Developer Certificate of Origin](https://developercertificate.org/)
to confirm that you have the right to submit your work. Every commit must
carry a `Signed-off-by:` line:

```
Signed-off-by: Your Name <your.email@example.com>
```

Add it automatically with `git commit -s`. By signing off you certify the
contents of <https://developercertificate.org/> — in short, that you wrote
the patch, or have the right to pass it on under the project's licence.

Pull requests that fail the DCO check will be blocked from merge until every
commit is signed off.

## How to contribute

1. Fork the repo and create a topic branch off `main` (`git switch -c
   fix/short-description`).
2. Make your changes. Keep diffs focused.
3. If you're adding behaviour, add or update tests.
4. Run the project's lint / test commands (see `README.md`).
5. Commit with `git commit -s -m "feat: short description"`.
6. Open a pull request against `insites-io/<repo>:main`. Describe the
   problem you're solving and how you've verified the fix.

## Code of Conduct

This project follows the [Contributor Covenant](CODE_OF_CONDUCT.md). By
participating you agree to abide by it.

## Security issues

Please do **not** open public issues for security vulnerabilities. Email
[security@insites.io](mailto:security@insites.io) instead, and we will
respond within five business days.
